# Prompt Examples

下面这些 prompt 可以直接复制到 Codex 里使用。

## 只做配图规划

```text
Use $eatmoreduck-illustrations 先不要生图。
请分析下面这篇文章哪里值得配图，输出 5 张左右的 shot list。
每张图写清楚：
- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 多吃鸭在图里做什么（吞 / 嚼 / 吐 / 滤等吃主题动作）
- 建议元素
- 建议中文标注词

<粘贴文章>
```

## 文章正文配图

```text
Use $eatmoreduck-illustrations 把下面这篇文章生成 4 张多吃鸭怪诞正文配图。
要求：16:9 横版、纯白背景、黑色手绘线稿、少量红橙蓝中文手写批注。
每张图只讲一个核心结构，不要做 PPT 信息图，不要可爱卡通。

<粘贴文章>
```

## 长文配图策略

```text
Use $eatmoreduck-illustrations 给这篇长文做配图策略。
不要平均配图，只挑认知锚点：核心判断、输入输出闭环、前后对比、常见坑、承接路径。
默认 6-8 张，先输出 shot list，不要生成图片。

<粘贴文章>
```

## 单个观点生成一张图

```text
Use $eatmoreduck-illustrations 为这个观点生成一张 16:9 正文配图：

信任不是喊出来的，而是一块证据一块证据铺过去。

画面要怪诞但清爽，多吃鸭必须承担核心动作。
中文标注最多 5 个，短一点。
```

## 强化橡皮鸭体块

```text
Use $eatmoreduck-illustrations 为“从手动 prompt 到自动化配图流水线”生成一张正文配图。
保留纯白背景、黑色手绘线稿、深灰/近黑鸭身、短宽橙色扁嘴、贴身圆侧翼和鸭子剪影。
不要画成鱼、潜艇、小鸡、手、猫爪或可爱小黄鸭贴纸。
```

## 工作流主题

```text
Use $eatmoreduck-illustrations 为“把一条原始素材加工成流量、信任、转化三种内容”生成一张图。
不要画正式流程图，不要复刻一鱼多吃旧案例。
请重新发明一个新的低科技隐喻，让多吃鸭用吃 / 消化 / 吞吐的方式参与核心动作。
```

## 改图：去掉标题

```text
Use $eatmoreduck-illustrations 帮我编辑这张图。
去掉左上角的“Workflow / 流程图”标题和下划线，其他内容保持不变。
不要新增任何文字或物件。
```

## 改图：增强多吃鸭参与感

```text
Use $eatmoreduck-illustrations 这张图方向对，但多吃鸭有点像装饰。
请保持核心意思不变，重生成一版：让多吃鸭真正在吃 / 消化 / 吐出推动结构运转的东西。
画面更怪一点，但仍然纯白、清爽、少字。
```

## 生成一组风格样片

```text
Use $eatmoreduck-illustrations 输出 5 个不同主题的多吃鸭正文配图效果。
主题分别覆盖：信息过载、产品验证、内容复利、一人公司、信任建立。
每张单独生成，不要拼成一张。
```
